## 1 设定窗口的管理逻辑

### 1.1 类似windows系统的任务管理器，列出所有进程

### 1.2 区分“应用”与“后台进程”：

Windows 任务管理器 “进程” 标签下的 **应用** 与 **后台进程**，核心区分规则只有一条：**是否有可见窗口（UI 线程）**。

| 类别                              | 判定标准                                                     | 典型特征                                  | 常见示例                                        |
| --------------------------------- | ------------------------------------------------------------ | ----------------------------------------- | ----------------------------------------------- |
| **应用 (App)**                    | 进程拥有**可见窗口**（有 UI 线程 / 主窗口句柄）Microsoft Learn | 可交互、可关闭、会在任务栏 / Alt+Tab 显示 | 浏览器、Office、微信、代码编辑器                |
| **后台进程 (Background Process)** | 进程**无可见窗口**（无 UI 线程）Microsoft Learn              | 无界面、常驻后台、用户不直接交互          | 浏览器渲染 / 插件进程、输入法、下载器、应用助手 |

### 补充说明

1. **边界情况**

   

   - 最小化到托盘的应用（如钉钉）仍算 “应用”，因主进程有窗口；托盘仅为子窗口 / 图标。
   - 部分应用拆分为 “前台 + 后台” 组合（如 Chrome 多进程），主窗口进程归 “应用”，渲染 / 插件归 “后台进程”。

   

2. **与 “Windows 进程” 的区别**

   

   - **Windows 进程**：系统关键进程（如 `explorer.exe`、`system`），标记为 critical，通常不建议结束Microsoft Learn。

   

3. **实用判断技巧**

   

   - 在任务管理器 “进程” 标签，勾选 **“命令行”** 列，查看进程是否有 `-window`、`-ui` 等启动参数辅助判断。
   - 结束 “应用” 可直接右键 “结束任务”；结束 “后台进程” 需谨慎，避免影响主程序或系统稳定。

### 1.3 对于"应用"进行管理，获取列表：

**完全能精准拿到任务管理器里显示的「应用」列表**（就是带窗口、出现在 Alt+Tab、任务栏里的那些）。

原理就是：**任务管理器判定 “应用” 的官方标准 = 有可见、可切换、非子窗口的顶层窗口**。

# Python 代码：获取任务管理器中的「应用」列表

依赖：`pywin32`

```
import win32gui
import win32process
import psutil

def is_visible_top_window(hwnd):
    """判断一个窗口是否是任务管理器认定的「应用窗口」"""
    # 1. 必须可见
    if not win32gui.IsWindowVisible(hwnd):
        return False

    # 2. 必须是顶层窗口（无父窗口）
    if win32gui.GetParent(hwnd) != 0:
        return False

    # 3. 必须有 WS_EX_APPWINDOW 或 WS_EX_WINDOWEDGE 风格（Alt+Tab 可切换）
    exstyle = win32gui.GetWindowLong(hwnd, win32con.GWL_EXSTYLE)
    if not (exstyle & win32con.WS_EX_APPWINDOW):
        return False

    # 4. 窗口标题不能为空
    title = win32gui.GetWindowText(hwnd).strip()
    if not title:
        return False

    return True

def get_taskmanager_app_list():
    app_list = []

    def callback(hwnd, extra):
        if is_visible_top_window(hwnd):
            title = win32gui.GetWindowText(hwnd)
            tid, pid = win32process.GetWindowThreadProcessId(hwnd)
            try:
                proc = psutil.Process(pid)
                app_list.append({
                    'pid': pid,
                    'name': proc.name(),
                    'title': title,
                    'exe': proc.exe()
                })
            except:
                pass
        return True

    win32gui.EnumWindows(callback, None)
    return app_list


# 测试
if __name__ == '__main__':
    import win32con
    apps = get_taskmanager_app_list()
    for app in apps:
        print(f"[{app['pid']}] {app['name']} - {app['title']}")
```

它**完全复刻任务管理器筛选 “应用” 的逻辑**：

1. 窗口必须 **可见**
2. 必须是 **顶层窗口**（不是子控件）
3. 必须是 **Alt+Tab 可切换** 的窗口（`WS_EX_APPWINDOW`）
4. 必须有 **窗口标题**
5. 排除托盘类后台小窗口

这样得到的列表，**和任务管理器 → 进程 → “应用” 一栏完全一致**。

4. WMI 事件监听机制

   - 软件运行期间，可能我们打开了一些进程，这时候，就需要WMI 事件监听机制来监测到这个进程启动了，然后设定窗口列表中的状态进行实时更新且热键可以对这个进程进行显示隐藏窗口的操作

import wmi
import threading

def monitor_process_creation():
    """实时监听新进程创建（专业方案）"""
    def watcher():
        c = wmi.WMI()
        # 监听 Windows 进程创建事件（系统级推送，不占CPU）
        process_watcher = c.Win32_Process.watch_for("creation")
        
        while True:
            try:
                # 阻塞等待，进程来之前不占CPU
                new_process = process_watcher()
                pid = new_process.ProcessId
                name = new_process.Name
                cmdline = new_process.CommandLine
                
                print(f"【新进程启动】pid={pid}, name={name}")
                
                # =============================================
                # 你在这里写判断：
                # if "xxx.exe" in name:
                #     do_something()

            except Exception as e:
                break

    # 后台线程运行，不卡主程序
    thread = threading.Thread(target=watcher, daemon=True)
    thread.start()

# 窗口管理器流程文档

## 1. 概述

窗口管理器是一个用于管理Windows系统窗口的应用程序，主要功能包括：
- 窗口枚举和分类
- 窗口隐藏和显示
- 热键管理
- 进程白名单管理

## 2. 核心组件

### 2.1 WindowManager (src/manager.py)
- 负责窗口枚举、缓存和状态管理
- 维护窗口列表（可见窗口、隐藏窗口）
- 处理窗口的隐藏和显示操作

### 2.2 UI主界面 (src/ui_main.py)
- 负责用户界面显示
- 窗口列表展示（选中窗口、可见窗口、后台应用）
- 热键录制和管理
- 关键字管理

### 2.3 配置管理 (src/config.py)
- 负责配置文件的加载和保存
- 存储选中窗口、隐藏窗口、进程白名单等信息

## 3. 窗口管理流程

### 3.1 窗口枚举流程

```
1. 用户触发刷新（手动或自动定时）
   ↓
2. WindowRefreshThread.run() 在后台线程执行
   ↓
3. window_manager.refresh_windows() 枚举系统窗口
   ↓
4. window_manager.get_cached_windows() 获取缓存的窗口列表
   ↓
5. 窗口分类（选中窗口、可见窗口、后台应用）
   ↓
6. 发送 refresh_finished 信号到主线程
   ↓
7. _on_refresh_finished() 更新UI表格
```

### 3.2 窗口分类逻辑

窗口分类基于以下规则：

1. **选中窗口**：
   - 在 `_selected_windows` 集合中的窗口
   - 无论是否可见，都添加到选中窗口列表
   - 包括被软件隐藏的窗口

2. **可见窗口**：
   - 不在 `_selected_windows` 集合中
   - 窗口可见（`is_visible = True`）
   - 是任务栏窗口（`is_taskbar_window` 返回 True）

3. **后台应用**：
   - 不在 `_selected_windows` 集合中
   - 窗口可见（`is_visible = True`）
   - 不是任务栏窗口

### 3.3 进程白名单机制

系统中有两种白名单：

#### 3.3.1 设置选项卡的进程白名单 (`config.process_whitelist`)
- **来源**：用户手动添加
- **存储**：config.json文件
- **作用**：最高优先级，这些进程不会被隐藏
- **管理方式**：只能通过"设置"选项卡手动添加/删除
- **用户要求**：程序不要自动添加，只能人工添加

#### 3.3.2 名单管理选项卡的白名单 (`window_manager._foreground_cache`)
- **来源**：程序自动识别
- **存储**：内存缓存（`_foreground_cache`）
- **作用**：具有窗口标题和句柄的前台应用进程
- **黑名单**：没有窗口标题的进程
- **管理方式**：程序自动识别，也可手动添加/删除

### 3.4 选中窗口管理

#### 3.4.1 选中窗口的添加
1. **从配置加载**：
   - 启动时调用 `_load_selected_windows_from_config()`
   - 根据config.json中的 `selected_windows` 列表
   - 通过进程名和标题查找当前运行的窗口
   - 将匹配的窗口句柄添加到 `_selected_windows` 集合

2. **用户手动添加**：
   - 从可见窗口或后台应用拖动到选中窗口表格
   - 右键菜单选择"选中窗口"
   - 双击窗口表格中的窗口

3. **自动选中**：
   - `_auto_select_keyword_windows()` 方法
   - 根据关键字自动选中匹配的窗口

#### 3.4.2 选中窗口的持久化
- 保存到config.json的 `selected_windows` 字段
- 格式：`[{"process_name": "book.exe", "title": "自定义工具栏"}]`
- 使用进程名和标题，而不是句柄（因为句柄会变化）

### 3.5 窗口隐藏和显示

#### 3.5.1 隐藏窗口
1. 用户触发隐藏操作（热键或按钮）
2. 对选中窗口列表中的窗口调用 `window_manager.hide_window(hwnd)`
3. 使用 `win32gui.ShowWindow(hwnd, win32con.SW_HIDE)` 隐藏窗口
4. 将窗口添加到 `_hidden_windows` 集合
5. 保存到config.json的 `hidden_windows` 字段

#### 3.5.2 显示窗口
1. 用户触发显示操作（热键或按钮）
2. 对隐藏窗口列表中的窗口调用 `window_manager.show_window(hwnd)`
3. 使用 `win32gui.ShowWindow(hwnd, win32con.SW_SHOW)` 显示窗口
4. 从 `_hidden_windows` 集合移除窗口
5. 从config.json的 `hidden_windows` 字段移除

## 5. 线程模型

### 5.1 主线程
- 负责UI更新和用户交互
- 处理窗口表格的显示
- 响应热键操作

### 5.2 后台线程 (WindowRefreshThread)

## 6. 常见问题和解决方案

### 6.1 选中窗口栏为空
**可能原因**：
1. `_selected_windows` 集合为空
2. 窗口枚举失败
3. UI更新逻辑问题
4. `_load_selected_windows_from_config()` 在窗口管理器启动前被调用

**解决方案**：
1. 检查config.json中是否有 `selected_windows` 配置
2. 检查 `_load_selected_windows_from_config()` 是否正确执行
3. 检查 `_update_window_table_incremental()` 是否正确更新表格
4. 将 `_load_selected_windows_from_config()` 延迟到第一次窗口刷新时调用

### 6.2 进程白名单自动添加
**问题**：程序自动添加进程到 `config.process_whitelist`
**解决方案**：
1. 移除 `ui.py` 中自动添加软件自身进程的逻辑
2. 确保只有用户手动添加的进程才会加入白名单
