# 一 优化方案

基于Windows窗口机制文档的最佳实践，提出以下优化方案：

1. ## 缓存设计优化

  实现分层缓存结构：
  第一层：永久黑名单（进程名），用于快速跳过已知的后台进程
  第二层：运行时白名单（进程名 → 多实例），存储当前运行的窗口信息
  使用进程名作为永久标识：进程名是唯一不变的标识，适合作为缓存键
2. ## 性能优化

  实现增量检测：
  首次启动时全量扫描，建立缓存
  后续只检测新出现的进程，跳过已知的后台进程
  黑名单机制：直接跳过系统关键进程和已知的后台进程
3. ## 隐藏窗口恢复机制

  实现特征识别：通过窗口可见性、窗口层级、窗口标题/类名等特征识别被隐藏的窗口
  自动恢复：工具重启后自动检测并具备恢复之前隐藏的窗口功能（通过热键或现有按钮）
4. ## 代码结构优化

  简化SafeWindowsAPI：优化Windows API调用，提高代码可读性
  模块化设计：将窗口管理功能拆分为多个模块，提高代码可维护性（不改变UI界面）
5. ## 其他优化

  缓存持久化：将黑名单和白名单缓存保存到文件，工具重启后直接加载

  名单管理：增加按钮，黑名单和白名单通过按钮点击，弹出列表，可以手动增加或删减列表中的项

  多显示器支持：完善多显示器场景下的窗口管理
  错误处理优化：增强错误处理机制，提高系统稳定性
